    Prosedural Programming : pemrograman secara prosedural

    Karakteristik Procedural Programming

    • Intruksi dilakukan langkah demi langkah
    • Memecah program menjadi bagian2 kecil
    • Disebut prosedur, subroutine / function
    • Linear / Top-to-Bottom : program dibaca dari atas kebawah
    • bahasa pemrograman menggunakan procedural : Fortran, ALGOL, COBOL, Pascal, C, PHP, Javascript

    Kelebihan Procedural Programming

    • TO-the-point : langsung melakukan apa yang kita mau dengan menuliskan baris2 tertentu
    • Simplicity & kemudahan implementasi (untuk compiler & interpreter) : kemudahan dibaca untuk compiler / interpreter
    • Mudah ditelusuri : karena linear mudah ditelusuri, kalo ada error pesan kesalahannya akan langsung menargetkan area kesalahannya
    • Membutuhkan lebih sedikit memory (dibandingkan dengan OOP) : karena dengan OOP, ada banyak hal yang harus diload duku kememory sebelum kita bisa mengerjakan sesuatu

    Karakteristik Object Oriented Programming

    • Menyusun semua kode program & struktur data sebagai objek
    • Objek adalah unit dasar dari program
    • Objek menyimpan data & perilaku : cenderung tidak punya variable & funtion, kita punya data / perilaku / behavior, yang kita sebut property & method
    • Objek bisa saling berinteraksi : kita bikin banyak objek dan objek itu akan saling berinteraksi
    • bahasa pemrograman : Java, Ruby, Python, C++, Javascript, PHP5

    Kelebihan Object Oriented Programmig

    • Representasi dunia nyata : memudahkan kita mempresentasikan kebutuhan kita didunia nyata
    • Enkapsulasi & Abstraksi Data
      • Enkapsulasi : Memungkinkan data yang kita buat aman, dari sisi tidak bisa diakses oleh bagian lain dari program kita
      • Abstraksi Data : Mudah untuk menyembunyikan kerumitan dari Program kita
    • Reusability : dapat dengan mudah untuk menggunaka ulang bagian2 dari program yang kita buat
    • Skalabilitas & Ekstensibilitas
      • memudahkan sipengembang / programmer, ketika nanti programnya akan menjadi lebih besar
      • kalo menggunakan procedural, lebih susah karena : semuanya biasanya disimpan dalam 1 tempat
    • Kemudahan pengelolaan
    • Kolaborasi : kalo bekerja dalam team, sudah banyak orang yang mengerjakan dalam 1 project, akan lebih mudah ketika kita membuat programnya menggunakan paradigma object oriented
    • Digunakan oleh framework

    Procedural VS. Object Oriented

    • semua mempunyai kelebihan / focus nya sendiri2
    • kalo misal ingin membuat programnya simple, dikerjakan sendiri : procedural
    • ketika programnya besar, dikerjakan banyak orang : object oriented
    • ketika ada yang bilang procedural itu jelek, gak aman : itu bukan salah dari paradigmanya, tapi kesalahan pengodingnya, mungkin ngodingnya gak bener

    Visibility

    Pengertian

    • == Access Modifier
    • » Sebuah konsep yang digunakan untuk mengatir akses dari property & method pada sebuah objek
    • Ada 3 keyword visibility :
      • public dapat digunakan dimana saja, bahkan diluar class
      protected hanya dapat digunakan di dalam sebuah class beserta turunannya
      private hanya dapat digunakan di dalam sebuah class tertentu saja

    Kenapa?

    • Hanya memperlihatkan aspek dari class yang dibutuhkan oleh client
    • Menentukan kebutuhan yang jelas untuk object
    • Memberikan kendali pada kode untuk menghindari bug

    Abstract Classs

    Pengertian

    • Sebuah kelas yang tidak dapat di-instansiasi
      • kita akan instansiasi kelas turunannya
    • kelas 'abstark'
      • karena tidak dapat diinstansiasi
      • yang menjadi kelas2 sesungguhnya adalah class2 turunannya
    • mendefinisikan interface untuk kelas lain yang menjadi turunnanya
    • Berperan sebagai 'kerangka dasar' untuk kelas turunnanya
      • kelas2 turunnanya akan bekerja sesuai dengan kerangka / interface yang sudah kita buat dikelas Abstract
    • Memiliki minimal 1 method abstrak
      • method ini yang akan kita anggap sebagai interface / kerangka dari method yang akan kita buat class2 turunannya
    • Digunakan dalam 'pewarisan' / inheritance untuk 'memaksakan' implementasi method abstrak yang sama untuk semua kelas turunnanya
      • didalam class abstrak punya sebuah method abstrak yang hanya interfacenya aja{namanya saja tidak ada isinya}, isinya nanti kita tuliskan diclass2 turunannya dengan menggunakan method yang sama
    • Semua kelas turunan, harus mengimplementasikan method abstrak yang ada di kelas abstraknya
    • Kelas abstrak boleh memiliki property / method reguler
    • Kelas abstrak boleh memiliki property / static method

    Study Casus

    • pasti bingung
    • jadi class buah ini akan kita buat menjadi abstrak

    Membuat kelas abstrak

    • abstract class Buah { }

    • abstract public function makan();
      • method abstrak

    Contoh Lain

    • class Mobil Extends Kendaraan
    • class Laptop Extends Komputer
    • class Email Extends Komunikasi
    • ...

    Alasan class abstract

    • Mempresentasikan ide / konsep dasar
    • "Composition over Inheritance"
      • sebaiknya kita melakukan composisi dibanding kita melakukan inheritance / pewarisan begitu saja
      • sebetulnya tanpa menggunakan class abstract bisa hanya menggunakan inheritance
      • tapi akan aneh ketika kita instansiasi class buah, padahal ketika class buah tidak kita instansiasi tidak ada masalah
      • composisi disini maksudnya : menggunakan abstract / interface
    • Salah satu cara menerapkan Polimorphism
    • Sentralisasi logic
    • Mempermudah pengerjaan tim
      • dengan class abstract, kita membuat class abstract yang disebut base class / class utama
      • didalam base class, ditulis beberapa method abstract
      • sehingga nanti kalo misal ada, rekan tim akan membuat sub-class / kelas turunan, mereka sudah tau, didalam class turunannya itu wajib ada sebuah method sebagai implementasi dari method abstactnya
